# תרגומים — OC Woo Shipping (`ocws`)

Text domain: **`ocws`**  
תיקייה: `wp-content/plugins/oc-woo-shipping/languages/`

## קבצים שצריכים להיות כאן

| קובץ | תפקיד | נטען באתר? |
|------|--------|------------|
| `ocws.pot` | תבנית מחרוזות (סריקת קוד) | לא |
| `ocws-he_IL.po` | עברית — עריכה ב-Loco | לא |
| `ocws-he_IL.mo` | עברית — **מה שהאתר קורא** | **כן** |
| `ocws-en_US.po` / `.mo` | אנגלית (אופציונלי) | MO בלבד |
| `update-i18n.sh` | סקריקה + קומפילציה | לא |

**אין צורך ב:** `*.l10n.php`, `*-backup-*.po~`, `languages-copy/`, `languages-2/`, `ocws.pot` בשורש התוסף.

## סריקת מילים חדשות (מתי ואיך)

### מתי להריץ סריקה

הרץ סריקה אחרי ש:

- הוספת או שינית מחרוזת בקוד עם `__()`, `_e()`, `esc_html__()`, `_x()`, `_n()` ו-domain **`ocws`**
- ערכת קבצי PHP בתבניות (`template-parts/`, `includes/`, `public/`)
- ערכת JS עם מחרוזות שעוברות דרך `wp.i18n` / `wp_localize_script` (רק אם ה-msgid מוגדר ב-PHP)

דוגמה בקוד:

```php
esc_html_e( 'Choose later', 'ocws' );
```

### מה הסריקה עושה (3 שלבים)

| שלב | פקודה | תוצאה |
|-----|--------|--------|
| 1 | `make-pot` | מעדכן `ocws.pot` — רשימת **כל** המחרוזות מהקוד |
| 2 | `update-po` | ממזג מחרוזות **חדשות** ל-`ocws-he_IL.po` / `ocws-en_US.po` בלי למחוק תרגומים קיימים |
| 3 | `make-mo` | מקמפל PO → **MO** (מה שהאתר קורא) |

### הרצה מהירה

מתיקיית התוסף (`wp-content/plugins/oc-woo-shipping`):

```bash
bash languages/update-i18n.sh
```

**Git Bash / WSL / שרת Linux** — דורש `wp` (WP-CLI) ב-PATH.

### הרצה ב-PowerShell (Windows)

```powershell
cd C:\path\to\wp-content\plugins\oc-woo-shipping

$EXCLUDE = ".git,node_modules,vendor,languages,.cursor,.claude,languages-copy,languages-2"

wp i18n make-pot . languages/ocws.pot `
  --slug=oc-woo-shipping `
  --domain=ocws `
  --package-name="Original Concepts WooCommerce Advanced Shipping" `
  --exclude=$EXCLUDE

wp i18n update-po languages/ocws.pot languages/
wp i18n make-mo languages/
```

בסיום אמור להופיע משהו כמו: `OK: ocws.pot, ocws-he_IL.po + .mo, ...`

### אחרי הסריקה — תרגום המילים החדשות

**אופציה א — Loco (מומלץ לעורך ויזואלי):**

1. Loco → Plugins → **Original Concepts WooCommerce Advanced Shipping**
2. פתח **Hebrew (he_IL)**
3. **Sync** (מושך מחרוזות חדשות מה-POT)
4. סנן **Untranslated** / חפש את המילה באנגלית
5. הזן תרגום עברי → **Save** → **Compile**

**אופציה ב — עריכה ידנית של PO:**

פתח `ocws-he_IL.po`, חפש מחרוזת עם `msgstr ""` (ריק) מתחת ל-msgid החדש:

```po
msgid "Choose later"
msgstr "בחר מאוחר יותר"
```

ואז:

```bash
wp i18n make-mo languages/
```

### זרימה מלאה: מילה חדשה בקוד → עברית באתר

```
1. כותבים בקוד:  __( 'New label', 'ocws' )
2. סריקה:       bash languages/update-i18n.sh
3. תרגום:       Loco (Sync → תרגום → Save → Compile)
                 או עריכת ocws-he_IL.po + wp i18n make-mo languages/
4. העלאה:       ocws-he_IL.po + ocws-he_IL.mo לשרת
5. באתר:        ניקוי cache + רענון קשיח
```

### טיפים

- בקוד תמיד **אנגלית** ב-msgid, עברית רק ב-PO/MO (לא להפך).
- אחרי Loco **Save** — ודא ש-`ocws-he_IL.mo` התעדכן (תאריך/גודל).
- אם `wp` לא מזוהה בטרמינל — התקן WP-CLI או הרץ מהתיקייה שבה WordPress מותקן עם `wp --path=...`.

## Loco Translate

1. **Plugins → Original Concepts WooCommerce Advanced Shipping → Hebrew (he_IL)**
2. **Sync** — מושך מחרוזות חדשות מה-POT
3. תרגם מחרוזות חסרות → **Save**
4. **Compile** — חייב ליצור `ocws-he_IL.mo` באותה תיקייה

**הגדרות Loco (חשוב):**

- מיקום: **Author** / בתוך התוסף (`languages/`)
- פורמט: **MO** (לא PHP בלבד)
- אחרי Save: אם אין MO מעודכן → `wp i18n make-mo languages/`

## אם מחרוזת נשארת באנגלית — רשימת בדיקה

### 1. קובץ MO על השרת

- קיים `languages/ocws-he_IL.mo`?
- תאריך עדכון **אחרי** ה-PO?
- גודל > 0?

### 2. שפת האתר

**הגדרות → כללי → שפת האתר** = עברית (`he_IL`).  
אם השפה `en_US` — ייטען `ocws-en_US.mo` (אנגלית).

### 3. קונפליקט נתיבים

התוסף טוען **רק** מ:

`plugins/oc-woo-shipping/languages/ocws-he_IL.mo`

אם יש עותק ישן ב-`wp-content/languages/plugins/ocws-he_IL.mo` — מומלץ למחוק אותו (עלול לבלבל בדיקות; הטעינה שלנו מתעלמת ממנו אחרי `unload_textdomain`).

### 4. Loco שמר רק PHP

אם נוצר `ocws-he_IL.l10n.php` בלי MO מעודכן:

```bash
wp i18n make-mo languages/
```

או מחק את ה-`.l10n.php` ועבוד רק עם PO + MO.

### 5. Cache

- ניקוי cache (תוסף / Cloudflare / LiteSpeed)
- רענון קשיח (Ctrl+F5)
- WooCommerce fragments — הסל/צ’יפ מתעדכן ב-AJAX; נסה סל ריק / חלון פרטי

### 6. הקוד משתמש ב-domain הנכון?

מחרוזות התוסף: `__( '...', 'ocws' )` — לא `woocommerce` / `deliz-short`.

דוגמה: כפתור "בחר מאוחר יותר" בפופאפ:

- קובץ: `template-parts/public/popup.php`
- msgid: `Choose later`
- domain: `ocws`

### 7. סנכרון PO

אחרי `make-pot`, הרץ `update-po` וודא שב-`ocws-he_IL.po` יש:

```po
msgid "Choose later"
msgstr "בחר מאוחר יותר"
```

## איך הטעינה עובדת (טכני)

`includes/class-oc-woo-shipping-i18n.php`:

1. `plugins_loaded` (10) + `init` (20)
2. טוען `languages/ocws-{locale}.mo`
3. אם WordPress 6.x השאיר `NOOP_Translations` — גיבוי: ייבוא MO ישירות ל-`$l10n['ocws']`

זה תיקן מקרה שבו התרגום היה בקובץ אבל `__()` החזיר אנגלית.
