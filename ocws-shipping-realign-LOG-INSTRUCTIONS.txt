OCWS — debug log: איסוף / יישור משלוח אחרי ריקון סל
==============================================

1) הפעלת הלוג
-------------
הוסיפו ב-wp-config.php (מעל השורה “That’s all, stop editing!”):

  define( 'OCWS_SHIPPING_REALIGN_LOG', true );

או (ב functions.php / תוסף mu בלבד):
  add_filter( 'ocws_shipping_realign_log_enabled', '__return_true' );

2) איפה נשמר הקובץ
-------------------
WooCommerce כותב ל:
  wp-content/uploads/wc-logs/ocws-shipping-realign-YYYY-MM-DD-xxxxxxxx.log

(מומלץ: לאחר בדיקה, להסיר את הקבוע או לבטל את ה-filter כדי שלא יימשך הלוג בפרודקשן)

במסך אדמין:
  ווקומרס > מצב > יומנים
בחרו “ocws-shipping-realign” או פתחו את קובץ ה-.log ישירות ב-FTP/קובצים

3) רצף פעולות לשחזור הבעיה (חזרו עליו פעמיים)
---------------------------------------------
- להתחיל גולש/חלון פרטי (או מחקו עוגיות האתר) כדי לתעד סשן מלא.

א)  להוסיף מוצר לסל
ב)  לבחור «איסוף עצמי», בחור סניף/שעה, לאשר את הפופא־א
ג)  להסיר את כל הפריטים כך שהסל נהיה ריק
ד)  להוסיף שוב מוצר (לפי אותה דרך שבה מופיעה אצלכם הבעיה: פופ־אפ / עמוד מוצר / וכו’)
ה)  לבדוק: האם מוצג «משלוח» במקום איסוף

4) לשלוח לתמיכה/מפתח
------------------------
- לצרף את קובץ ה־.log שכותרתו מתחילה ב- ocws-shipping-realign
- לרשום בקצרה: דפדפן, אם היה לוג־אין למשתמש, ואם המחיקה הייתה ממיני־קארט או מעמוד עגלה

5) אירועים בלוג (מילות מפתח)
-----------------------------
- remove_cart_item_last_line — הוסר הפריט האחרון (לפני unset)
- merge_lost_chosen_shipping — שחזרנו rate מהגיבוי הקודם
- persist_backup — נשמר גיבוי + ocws_pending_shipping_realign
- realign_enter / realign_apply / realign_noop — ניסיון יישור אחרי הוספה מחדש
- restore_empty_cart_apply — שחזור session בסל ריק
- popup_save_* — אחרי שמירת פופ־אב כשהסל ריק/לא
